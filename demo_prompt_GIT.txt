[SLIDE: Impact Numbers]
"Let's talk outcomes."

Detection speed: From 24-hour batch cycles to 2-3 second real-time detection — that's a 99.99% reduction in detection latency.
False positive reduction: ML scoring (AUC > 0.95) combined with rule-based detection means analysts get a prioritized, explainable queue — estimated 60% fewer false positives reaching human review.
Investigation time: The Data Agent eliminates manual SQL querying. Estimated reduction from 45 minutes per case to under 8 minutes — the agent pulls history, flags, watchlist, scores, and recommends action in one response.

Regulatory readiness: Every flag carries full evidence chain — rule_id, condition_summary, rationale, ml_score, timestamps. SAR summaries are generated by the agent in regulator-ready format referencing BSA, FATF, and FIU-IND standards.
Learning loop: Analyst CONFIRMED/FALSE_POSITIVE decisions feed back into the next ML training run. False positives fall over time — the system gets smarter with use.

[SLIDE: Scalability]
"Scalability is built into the architecture, not bolted on."

Fabric's auto-scaling compute handles 10x transaction volume without architecture changes.
KQL rules are parameterized — add a new detection pattern by adding a new KQL query. No code deployment needed.
The Ontology is entity-driven — add a new entity type (Merchant, Branch) by binding a new table. The graph extends automatically.
Multi-region: Same architecture works across geographies with localized thresholds and country-specific watchlists.

[SLIDE: Adaptability — Cross-Industry]

Insurance: Swap KQL rules for claims fraud patterns (staged accidents, phantom injuries). Same agents, same graph.
Healthcare: Billing fraud, upcoding, phantom services — same structuring detection logic.
Telecom: SIM swap fraud, account takeover — same velocity and dormant-spike rules.
E-commerce: Payment fraud, refund abuse — same mule account and layering detection.
Government: Benefits fraud, procurement fraud — same relationship graph analysis.

"The AI Agent pattern is domain-agnostic. Swap the ontology, swap the KQL rules, and it works for any anomaly detection use case."

"Let me be clear about what we did differently."
1️⃣ Ontology-Driven Network Detection
"Traditional AML systems check transactions in isolation — flat rows in a table. We used Fabric IQ Ontology to model money-flow networks. The SENDS_TO relationship between Transaction and Account via counterparty_account creates a directed graph. Mule hubs, layering chains, and PEP connections become visible as graph patterns — something impossible in any flat-table system."
2️⃣ Dual-Agent Architecture — Investigation + Autonomous Operations
"We didn't just build a dashboard. We built TWO AI agents. The Data Agent lets investigators ask questions in plain English and get explainable, evidence-backed answers with recommended actions. The Operations Agent is an autonomous 24/7 watcher that polls, enriches with history and ML scores, escalates based on severity policy, and auto-opens cases — no human in the loop for the first response."
3️⃣ End-to-End Microsoft-Native — Zero Third-Party Dependencies
"Every component — Fabric, Eventhouse, OneLake, Power BI, Entra, Power Automate, Teams — is native Microsoft. No Kafka, no Splunk, no third-party AML vendor.
